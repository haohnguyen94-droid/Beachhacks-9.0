STEP 3: COPY AGENT ADDRESSES INTO .env

From the agent startup logs, copy:
  - Orchestrator address → ORCHESTRATOR_ADDRESS=
  - News Scraper address → SENTIMENT_AGENT_ADDRESS= (note: news scraper sends to sentiment)
  - Sentiment Agent address → SIGNAL_ENGINE_ADDRESS= (sentiment sends to signal engine)
  - Signal Engine address → DASHBOARD_URL= (signal engine sends to dashboard)

Example .env:
ORCHESTRATOR_ADDRESS=agent1qf0s8fkfkwu9dgnqrzhenysex6f0khvnpp5vr09acwzw4t9e7yjq...
SENTIMENT_AGENT_ADDRESS=agent1qwmltvjxte04e380qjuv3mdhkgdlgpc6ddw8vmf3eqfpzp5gjkzh...
SIGNAL_ENGINE_ADDRESS=agent1qXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX...
DASHBOARD_URL=http://localhost:8080

TROUBLESHOOTING

❌ "Agents can't communicate"
✅ Copy their printed addresses to .env, restart them

❌ "FinBERT takes 60+ seconds"
✅ Normal on first run, then cached

❌ "No signals appearing"
✅ Make sure all 6 services are running, check logs for errors

❌ "NewsData API rate limited"
✅ System auto-rotates to next API key (you have 3 keys in .env)

───────────────────────────────────────────────────────────────────────────────

QUICK TIPS

• Each agent needs its own terminal
• Agent addresses are printed at startup - copy them to .env
• Signal Engine must know FastAPI address (DASHBOARD_URL in .env)
• First FinBERT load takes 60s, then cached
• Check Terminal 5 (FastAPI) logs to see signals arriving
